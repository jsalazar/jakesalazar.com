Search & Filter UI

This is a psuedo-headless CMS implementation of a documents and forms area of the School of Education website. It's not fully headless, because it is not API driven--I simply wrote queries against the Expression Engine CMS backend to extract the data and provide it with a new interface. The entire UI is driven by JQuery. I also used .htaccess to work with PHP and JQuery to dynamically display individual file pages.

100% responsible for design and development

Multi-step Applications

I built dozens of these each with varying complexity and requirements supporting multi-step workflows. The entire application was generated by a program I wrote called FormBot. FormBot is a JSON based form generator with CRUD capabilities. Essentially, a developer can fill out a JSON config file and FormBot practically does the rest. Apps can be deployed in a few minutes or at most in one hour if further customizations are needed.

My role:
100% responsible for design and development
Technology:
PHP, SLIM API Service, LDAP, MySQl, FormBot, JSON, HTML5, CSS3, JQuery, Bootstrap, Underscore.js
